初使Spring大礼包

前言

最近项目有需求想做聚合支付，看了好久的支付文档之后，就在网上找相关的源码，果然，php这种源码倒是挺开放的，中间穿插着几个Java和oc之类的，有想法学习Java，就拿个代码观察下，发下Spring大礼包果然用的还是比较多的。

这个大框架在之前就听说过，只是没想法去鼓捣，毕竟当时还属于事比较繁杂的时候，现在慢慢工作适应下来之后，白天看代码，晚上调代码，这日子也是醉了。

开发工具

Idea比eclipse好用的不止一星半点儿

参考着小娃娃(别问我这是谁)给的博客上面的资料看了下，创建maven工程就可以了，然后搞一个server一个client，然而没想到，在创建之后就已经红艳艳了，折腾了好长时间不知道是因为啥，感觉那时候以为自己眼花，把代码发给二哈(也别问我这是谁)在他那正常，发给三哈(-_-|||)也是正常，真是奇了怪了。

然后用了以下几种方法：

  1. Java环境重新配置
  2. 代码删除重新参考写
  3. 代码清理缓存重新打开 (File -> Invalidate Caches / Restart)
  4. maven配置文件重新导入 (右键工程 -> Maven -> Reimport)

试了这几个方法之后还是不行，把pom文件里面的版本号改了下(之前是2.0.1)，修改了之后看到了久违的后台下载进度条，感觉看到了希望的样子，就去搞了个蓝灯下好，之后果然可以了，看来还是配置的问题。

又一个问题

参考着继续往下写，看到里面有多个pom文件，可以做继承关系，参考着一搞，又提示找不到spring这个包，又是一个坑的出现，这次鼓捣的时间更长，参考上面的又做一遍之后发现没有效果，最后鬼使神差的找到了这个。

File -> Setting

没错，这个地方之前是3现在换成2就可以了，整个工程全都没红的了，当时的心情啊，真想砸电脑删库跑路（扯淡）。

折腾，总会有效果的
